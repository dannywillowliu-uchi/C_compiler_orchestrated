# C Compiler (Autonomously Built)

A functioning C compiler built from scratch in Python by autonomous AI agents. Takes C source code as input and produces x86-64 assembly output. No human wrote any compiler code -- every line was generated by [autodev](https://github.com/dannywillowliu-uchi/autonomous-development).

```
C source -> Preprocessor -> Lexer -> Parser -> Semantic Analysis -> IR Gen -> IR Optimizer -> Codegen -> Peephole -> x86-64 Assembly
```

## Usage

```bash
git clone git@github.com:dannywillowliu-uchi/C_compiler_orchestrated.git
cd C_compiler_orchestrated
uv sync --extra dev

# Compile a C file to x86-64 assembly
PYTHONPATH=src .venv/bin/python -m compiler input.c -o output.s

# Run the test suite
PYTHONPATH=src .venv/bin/python -m pytest tests/ -q
```

## Example

```c
// fibonacci.c
int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

int main() {
    return fibonacci(10);
}
```

```bash
PYTHONPATH=src .venv/bin/python -m compiler fibonacci.c -o fibonacci.s
```

Produces valid x86-64 AT&T syntax assembly with System V ABI calling convention.

## What It Supports

**Types**: `int`, `char`, `void`, `float`/`double`, `unsigned`, pointers, arrays, structs, unions, enums, typedefs, `const`/`volatile`/`static`/`extern`, cast expressions

**Expressions**: arithmetic, comparison, logical (`&&`/`||` with short-circuit), bitwise, assignment (`=`, `+=`, `-=`, etc.), `++`/`--`, ternary, `sizeof`, comma, function calls (direct + function pointers), array subscript, struct/union member access (`.` and `->`)

**Statements**: `if`/`else`, `while`, `for`, `do`-`while`, `switch`/`case`/`default`, `break`, `continue`, `return`, compound blocks

**Preprocessor**: `#define` (object + function macros), `#ifdef`/`#ifndef`/`#if`/`#elif`/`#else`/`#endif`, `#include`, `#undef`, `#error`, `#warning`, `#line`, `#pragma once`, token pasting (`##`), stringification (`#`)

**Optimizations**: constant folding, copy propagation, DCE, CSE, LICM, strength reduction, jump threading, unreachable code elimination, peephole patterns (mov elimination, LEA folding, dead store removal), graph-coloring register allocator

**Codegen**: x86-64 AT&T syntax, System V AMD64 ABI, SSE for floating-point, global/static data sections, string literal pooling

## How It Was Built

This compiler was built by [autodev](https://github.com/dannywillowliu-uchi/autonomous-development), an autonomous development framework that orchestrates parallel Claude Code workers.

The process:
1. We provided an objective ("build a C compiler") and a verification command (`pytest && ruff`)
2. autodev's planner decomposed the work into parallel units
3. 4 Claude Code workers executed units in isolated git worktrees
4. Results merged via a green-branch strategy, replanning each epoch

| Metric | Value |
|--------|-------|
| Wall time | ~5 hours |
| API cost | ~$55 |
| Workers | 4 parallel |
| Units merged | ~35 |
| Source code | 8,106 lines (16 modules) |
| Test code | 22,309 lines (42 test files) |
| Tests passing | 1,788 |
| Human-written compiler code | 0 lines |

See [docs/checkpoint-1.md](docs/checkpoint-1.md) for detailed breakdown.

## Project Structure

```
src/compiler/
    __main__.py        # CLI entry point
    tokens.py          # Token type definitions
    lexer.py           # C tokenizer
    preprocessor.py    # Preprocessor directives + macro expansion
    ast_nodes.py       # AST node types
    parser.py          # Recursive descent parser with precedence climbing
    semantic.py        # Type checking + symbol table
    ir.py              # Three-address code IR definitions
    ir_gen.py          # AST -> IR lowering
    optimizer.py       # IR optimization passes (constant fold, DCE, CSE, LICM)
    cfg.py             # Control flow graph construction
    liveness.py        # Liveness analysis
    regalloc.py        # Graph-coloring register allocator
    codegen.py         # IR -> x86-64 assembly
    peephole.py        # Assembly peephole optimizer
```

## Continuing Development

The compiler can be extended further by re-running autodev:

```bash
cd /path/to/autonomous-development
uv run mc mission --config /path/to/C_compiler_orchestrated/autodev.toml
```

The planner will survey the codebase, identify gaps, and dispatch workers to fill them.
